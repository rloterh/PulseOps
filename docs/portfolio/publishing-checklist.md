# PulseOps Publishing Checklist

Use this before sharing PulseOps in GitHub, job applications, Upwork, FlexJobs, or a personal portfolio.

## Repository

- [ ] `dev` includes the final Sprint 12 branch after review.
- [ ] README explains setup, architecture, demo seed flow, and tradeoffs.
- [ ] `.env.example` contains placeholders only.
- [ ] Demo credentials are clearly marked as local/reviewer-only.
- [ ] Screenshots do not expose local host tokens, Supabase keys, Stripe keys, or private account details.

## Demo

- [ ] The first reviewer flow starts with `owner@pulseops-demo.com`.
- [ ] Branch comparison has visible differences across all branches.
- [ ] Jobs and incidents include overdue, at-risk, blocked, resolved, and completed examples.
- [ ] Billing is shown as realistic state, not a fake live payment claim.
- [ ] AI is described as assistive and explainable, not authoritative.

## Application Blurbs

Use the short version when space is limited:

> Built PulseOps, an AI-powered operations command center for multi-location service businesses using Next.js, TypeScript, Supabase, Stripe, and a deterministic AI summary layer. The platform includes workflow management, analytics, incidents, billing, auditability, hardening, and portfolio-grade packaging.
